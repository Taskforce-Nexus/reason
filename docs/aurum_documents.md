# AURUM Document System

AURUM generates structured venture documentation and build artifacts across three layers:

- Business
- Product
- Engineering

These outputs are not only for analysis. Together they transform a founder's idea into something that can be evaluated, designed, and built.

---

## Business Layer

These documents define the business logic and strategic foundation of the venture.

### 1. Value Proposition

Defines:

- ICP
- customer problem
- proposed solution
- value hypothesis
- product-market fit evaluation

---

### 2. Business Model

Defines:

- revenue model
- pricing structure
- channels
- acquisition logic
- unit economics

---

### 3. Customer Journey

Defines:

- discovery
- evaluation
- activation
- retention
- expansion

---

### 4. Branding

Defines:

- mission
- vision
- values
- positioning
- messaging
- naming
- identity direction

---

### 5. Business Plan

Defines:

- strategic direction
- venture readiness
- key risks
- opportunity framing
- go-to-market logic

---

## Product Layer

These artifacts convert the venture into a product system.

### 6. Product Concept

Defines:

- what the product is
- who it serves
- what problems it solves
- the product scope at a high level

---

### 7. PRD

Defines:

- product requirements
- expected behaviors
- feature intentions
- implementation constraints

---

### 8. UX Architecture

Defines:

- main user flows
- navigation structure
- screen relationships
- interaction architecture

---

### 9. Default Frames Inventory

Defines:

- approved default UI frames
- canonical initial product surfaces
- source-of-truth frame list before state expansion

---

### 10. Frame Expansion

Defines:

- derived UI states
- empty states
- loading states
- error states
- success states
- other necessary variants

---

### 11. Frame Scaffolding

Defines:

- frame purpose
- visible sections
- fields
- actions
- data shown
- component candidates
- implementation notes

---

### 12. Design System

Defines:

- design tokens
- colors
- typography
- spacing
- reusable UI components

---

## Engineering Layer

These artifacts prepare the product for implementation.

### 13. System Design

Defines:

- technical architecture
- services
- APIs
- data models
- integrations
- infrastructure direction

---

### 14. Backlog

Defines:

- epics
- features
- implementation tasks
- engineering priorities

---

### 15. Repo Blueprint

Defines:

- repository structure
- output packaging
- file layout for handoff
- build-ready project organization

---

## Summary

AURUM currently recognizes 15 canonical documents and artifacts:

- Business Plan

### Engineering (3)
